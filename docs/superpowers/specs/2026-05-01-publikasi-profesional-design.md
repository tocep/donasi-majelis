# Publikasi Profesional Design

## Ringkasan

Tahap 4 mempersiapkan website donasi agar siap dipublikasikan secara profesional. Website tetap dipertahankan sebagai situs statis karena struktur saat ini sudah cocok untuk GitHub Pages.

## Keputusan

- Gunakan Git repository lokal sebagai dasar versioning.
- Gunakan GitHub Pages sebagai hosting rekomendasi.
- Tambahkan `.nojekyll` agar GitHub Pages menyajikan file statis apa adanya.
- Tambahkan dokumentasi publikasi untuk panitia.
- Tambahkan analytics ringan yang nonaktif secara default.

## Batasan

- Tidak membuat remote GitHub karena membutuhkan akun dan nama repository dari pemilik proyek.
- Tidak mengaktifkan analytics pihak ketiga sebelum domain dan layanan dipilih.
- Tidak memasang analytics di halaman admin.

## Kriteria Selesai

- Repository lokal sudah bisa diinisialisasi dengan Git.
- File publikasi GitHub Pages tersedia.
- Panduan publikasi tersedia.
- Halaman publik memuat analytics no-op tanpa error.
- Website tetap bisa dibuka secara lokal.
